# Trippo（旅啵）产品功能说明

> 版本：V2.0  
> 更新日期：2026-07-11

## 一、产品概述

Trippo（旅啵）是一款移动端优先的旅行助手 App，帮助用户规划旅行、管理心愿单、记录打卡和撰写旅行手账。采用暖橙主色调 + 纯白背景的设计风格，提供温暖、舒适的使用体验。

**技术栈**：React 18 + Vite 5 + TypeScript 5 + Zustand 4 + React Router v6  
**数据存储**：localStorage 本地持久化，key 前缀 `trippo_`

---

## 二、底部导航架构

App 采用固定底部导航栏，包含 3 个主 Tab：

| Tab | 入口路径 | 功能说明 |
|-----|---------|---------|
| 行程 | `/trips` | 管理所有旅行行程，查看行程详情、规划路线等 |
| 心愿单 | `/wishlist` | 全局心愿单，收藏想去的景点、美食、购物地点等 |
| 旅行手账 | `/journals` | 全局手账列表，查看和管理所有旅行手账 |

## 三、行程模块

### 3.1 行程列表页（`/trips`）

**页面文件**：`src/pages/trip/TripListPage.tsx`

#### 功能列表：

1. **行程状态分区展示**
   - 进行中（ongoing）：当前时间在行程起止日期之间
   - 计划中（upcoming）：行程尚未开始
   - 已完成（completed）：行程已经结束
   - 每个分区独立展示对应的行程卡片

2. **状态筛选功能**
   - 右上角漏斗图标按钮，点击弹出筛选面板
   - 支持筛选：全部 / 进行中 / 计划中 / 已完成
   - 筛选后只展示对应状态的行程

3. **新建行程入口**
   - 底部浮动圆形橙色按钮（+），位置醒目
   - 点击跳转到新建行程页

4. **行程卡片展示**
   - 封面图片：优先显示行程封面图 → 关联景点图片 → 目的地默认图
   - 目的地名称 + 行程标题
   - 行程日期（起止日期）
   - 行程天数
   - 心愿数量统计

5. **状态自动流转**
   - 页面加载时自动调用 `checkAndUpdateStatus()`
   - 计划中 → 进行中：当前时间到达开始日期
   - 计划中/进行中 → 已完成：当前时间超过结束日期

6. **手动修改状态**
   - 点击行程卡片右上角菜单按钮
   - 支持手动切换到：计划中 / 进行中 / 已完成
   - 手动修改后，自动流转逻辑仍然生效

### 3.2 创建/编辑行程页（`/trips/create`、`/trips/:tripId/edit`）

**页面文件**：`src/pages/trip/CreateTripPage.tsx`

#### 功能列表：

1. **新建行程**
   - 输入行程标题
   - 选择目的地
   - 选择开始日期和结束日期
   - 可选：添加行程描述
   - 新建后自动跳转到行程详情页

2. **编辑行程**
   - 通过 URL 参数 `tripId` 判断是否为编辑模式
   - 可修改：标题、目的地、起止日期、描述
   - 保存后返回到行程详情页

3. **表单验证**
   - 标题不能为空
   - 结束日期不能早于开始日期

### 3.3 行程详情页（`/trips/:tripId`）

**页面文件**：`src/pages/trip/TripDetailPage.tsx`

#### 功能列表：

1. **行程头部信息**
   - 封面图（渐变背景 + 目的地首字 / 图片）
   - 行程标题和目的地
   - 行程日期和天数
   - 行程状态标签（进行中/计划中/已完成）

2. **快捷入口（4个）**
   - 心愿单：跳转到该行程的心愿单
   - 每日路线：跳转到每日路线规划页
   - 打卡：跳转到打卡列表页
   - 手账：跳到手账列表页（支持在列表页内新建手账）

3. **行程统计数据**
   - 心愿数量
   - 打卡数量
   - 手账数量
   - 总花费（基于打卡记录）

4. **右上角菜单**
   - 编辑行程：跳转到编辑页面
   - 删除行程：删除当前行程（含确认弹窗）
   - 修改状态：手动切换行程状态

## 四、心愿单模块

### 4.1 全局心愿单页（`/wishlist`）

**页面文件**：`src/pages/spot/GlobalWishlistPage.tsx`

#### 功能列表：

1. **搜索功能**
   - 支持按心愿名称搜索
   - 支持按地点/地址搜索（如搜索"浙江"会返回"西湖"等相关景点）
   - 实时搜索，输入即过滤

2. **心愿分类展示**
   - 按分类分组展示：景点 / 美食 / 购物 / 公园 / 乐园 / 文化 / 其他
   - 每个分类下展示对应的心愿卡片

3. **心愿卡片**
   - 心愿名称
   - 地址（如有）
   - 分类标签
   - 收藏状态指示

4. **加入行程功能**
   - 点击心愿卡片右侧"+"按钮
   - 弹出行程选择列表（BottomSheet）
   - 选择目标行程后，将心愿移入该行程的心愿单
   - 显示成功 Toast 提示

5. **心愿详情**
   - 点击心愿卡片进入详情页

### 4.2 行程心愿单页（`/trips/:tripId/spots/wishlist`）

**页面文件**：`src/pages/spot/WishlistPage.tsx`

#### 功能列表：

1. **行程内心愿管理**
   - 展示当前行程下的所有心愿
   - 按分类分组展示
   - 支持添加心愿到行程（搜索添加）

2. **安排到日程**
   - 支持将心愿安排到具体某一天
   - 支持从日程中移除（返回心愿池）
   - 支持调整当天内的顺序（上移/下移）

### 4.3 心愿详情页（`/wishlist/:spotId`、`/trips/:tripId/spots/:spotId`）

**页面文件**：`src/pages/spot/SpotDetailPage.tsx`

#### 功能列表：

1. **基础信息展示**
   - 封面图片
   - 心愿名称
   - 地址信息
   - 评分（如有）
   - 分类标签

2. **分类差异化展示**
   - 景点类：展示景点介绍、开放时间、门票信息
   - 美食类：展示推荐菜品、人均消费、营业时间
   - 购物类：展示主营类型、消费水平、营业时间
   - 公园/乐园类：展示游玩项目、建议游玩时长、门票信息
   - 文化类：展示文化背景、参观须知、开放时间

3. **操作按钮**
   - 收藏/取消收藏
   - 加入行程 / 移动到其他行程
   - 删除心愿

### 4.4 景点搜索页（`/trips/:tripId/spots/search`）

**页面文件**：`src/pages/spot/SpotSearchPage.tsx`

#### 功能列表：

1. **搜索景点**
   - 输入关键词实时搜索
   - 搜索结果包括：行程内已有景点 + 系统预置景点库

2. **添加到行程**
   - 点击搜索结果可直接添加到当前行程的心愿单

### 5.1 每日路线页（`/trips/:tripId/route`）

**页面文件**：`src/pages/route/RoutePage.tsx`

#### 功能列表：

## 十、数据模型

### 10.1 Trip（行程）

```typescript
interface Trip {
  id: string
  title: string           // 行程标题
  destination: string     // 目的地
  startDate: string       // 开始日期
  endDate: string         // 结束日期
  coverImage?: string     // 封面图片
  status: 'upcoming' | 'ongoing' | 'completed'  // 行程状态
  createdAt: string
  updatedAt: string
  description?: string    // 行程描述
}
```

### 10.2 Spot（景点/心愿）

```typescript
interface Spot {
  id: string
  tripId: string          // 所属行程
  name: string            // 名称
  address?: string        // 地址
  lat?: number            // 纬度
  lng?: number            // 经度
  category?: string       // 分类
  coverImage?: string     // 封面图
  rating?: number         // 评分
  wishlisted: boolean     // 是否在心愿单
  addedAt?: string        // 添加时间
  dayIndex?: number       // 安排到第几天
  orderInDay?: number     // 当天顺序
}
```

### 10.3 CheckIn（打卡）

```typescript
interface CheckIn {
  id: string
  tripId: string          // 所属行程
  spotId: string          // 关联景点
  date: string            // 打卡日期
  photos: string[]        // 照片
  notes?: string          // 备注
  cost?: number           // 花费
  category?: CheckInCategory  // 分类
  createdAt: string
}
```

### 10.4 Journal（手账）

```typescript
interface Journal {
  id: string
  tripId?: string         // 关联行程（可选）
  title: string           // 标题
  content: string         // 正文内容
  coverImage?: string     // 封面图
  createdAt: string
  updatedAt: string
  template?: string       // 模板
  status?: 'draft' | 'published'  // 状态
}
```

### 10.5 List（主题清单）

```typescript
interface List {
  id: string
  name: string            // 清单名称
  type: ListType          // 清单类型
  icon?: string           // 图标
  description?: string    // 描述
  items: ListItem[]       // 清单项
  createdAt: string
  updatedAt: string
}
```

## 十一、状态管理

使用 Zustand 进行状态管理，按领域划分为多个独立 store：

| Store | 文件路径 | 管理内容 |
|-------|---------|---------|
| useTripStore | `src/stores/useTripStore.ts` | 行程数据、状态流转 |
| useSpotStore | `src/stores/useSpotStore.ts` | 景点/心愿数据、搜索、排序 |
| useCheckInStore | `src/stores/useCheckInStore.ts` | 打卡记录 |
| useJournalStore | `src/stores/useJournalStore.ts` | 手账数据、草稿生成 |
| useListStore | `src/stores/useListStore.ts` | 主题清单数据 |

所有 store 均支持：
- localStorage 本地持久化
- 首次加载无数据时自动注入 mock 数据
- CRUD 操作
- 自动同步到本地存储
