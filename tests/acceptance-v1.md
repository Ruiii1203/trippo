# Trippo V1 验收文档

这份文档用于记录 V1.0 核心闭环的验收结果。

## 当前范围

V1.0 只关注以下能力：

- 创建行程
- 景点搜索与心愿清单
- 每日路线规划
- 景点打卡
- 手账草稿生成与查看

## 验收项

### 1. 创建行程

- [x] 能创建一条新行程
- [x] 目的地和日期能正确保存
- [x] 创建完成后进入行程详情

### 2. 景点与心愿清单

- [x] 能搜索并添加景点
- [x] 景点能进入心愿清单
- [x] 景点能被加入某一天路线

### 3. 每日路线

- [x] 能查看某一天的路线
- [x] 能调整景点顺序
- [x] 能保存路线

### 4. 打卡

- [x] 能在路线中进入打卡
- [x] 能保存照片 / 花费 / 感受
- [x] 打卡后能回到行程内查看

### 5. 手账

- [x] 能基于已有素材生成草稿
- [x] 能查看手账
- [x] 能保存手账修改

#### 回归验收总结

**结论：V1.0 可交付。** 五大模块完整闭环跑通，数据持久化正常，移动端适配到位，边界输入有基础校验。

**可优化项汇总（非阻断，建议后续迭代处理）：**

1. **心愿单「加入路线」入口**：当前点击后显示「功能开发中」toast，建议接通到每日路线页或移除，避免用户困惑。
2. **首次启动空状态**：清空数据后立即注入 mock，无法体验从零开始的真实空态。可考虑增加「清空体验数据」入口或首次启动引导。

**总体评分：** 功能完整性 ★★★★★ / 交互流畅度 ★★★★☆ / 边界健壮性 ★★★★☆
