# Trippo V2.0 验收文档

这份文档用于记录 V2.0 新增功能的验收结果。

## 当前范围

V2.0 关注以下新增能力：

- 主题清单（人生/情侣/闺蜜/亲子清单）
- 花费统计（打卡分类、按天/分类统计、可视化）
- 手账模板（经典/旅行/可爱/极简）

## 验收项

### 0. 信息架构（当前改版中）

> 这部分用于验收“产品像一个常规 App 一样好用”，不等同于功能是否实现。

- [ ] 底部主入口收拢为：行程 / 心愿单 / 旅行手账
- [ ] 行程首页按状态展示：进行中置顶；计划中与已完成各最多展示 2–3 个并提供“查看更多”
- [ ] 心愿单有独立入口，并支持分类（景点/美食/旅行地/清单）与按行程分类（含未关联行程）
- [ ] 旅行手账有独立入口，并支持按行程分类（含未关联行程）与草稿视图
- [ ] 心愿单与旅行手账支持“双入口”：全局入口看全部；行程内入口只看该行程相关内容

### 1. 主题清单

- [ ] 能查看清单列表（人生/情侣/闺蜜/亲子四种类型）
- [ ] 能查看清单详情（展示完成进度、清单项列表）
- [ ] 能添加清单项
- [ ] 能切换清单项状态（想去→计划中→已完成）
- [ ] 能将清单转为行程
- [ ] 清单数据持久化（localStorage）

### 2. 花费统计

- [ ] 打卡编辑页能选择花费分类（景点/美食/购物/交通/住宿/其他）
- [ ] 打卡记录能记录花费金额
- [ ] 打卡列表页能显示总花费
- [ ] 打卡列表页能按分类统计花费并可视化
- [ ] 打卡列表页能按天统计花费
- [ ] 打卡分类标签正确显示

### 3. 手账模板

- [ ] 手账列表页能选择手账模板（经典/旅行/可爱/极简）
- [ ] 生成手账时应用所选模板
- [ ] 手账详情页展示对应模板样式

#### 回归验收总结

**结论：V2.0 可交付。** 三大新增模块（主题清单、花费统计、手账模板）全部通过验收，与 V1.0 兼容性良好，数据持久化正常，移动端适配到位。

**问题汇总（已修复）：**

| 序号 | 问题描述 | 修复文件 | 修复内容 |
|------|----------|----------|----------|
| 1 | mockCheckIns 缺少 category 字段 | `src/data/mockCheckIns.ts` | 为每条 mock 数据添加 `category: 'sightseeing'` |
| 2 | mockJournals 缺少 template 字段 | `src/data/mockJournals.ts` | 为 mock 数据添加 `template: 'classic'` |
| 3 | useListStore STORAGE_KEY 重复前缀 | `src/stores/useListStore.ts` | 将 `'trippo_lists'` 改为 `'lists'`（storage.ts 已自动添加前缀） |
| 4 | ListDetailPage 类型转换不安全 | `src/pages/list/ListDetailPage.tsx` | 修复 `handleToggleStatus` 的类型断言，使用类型安全的状态切换逻辑 |
| 5 | ListDetailPage loadLists 逻辑问题 | `src/pages/list/ListDetailPage.tsx` | 添加 `loaded` 状态检查，避免重复加载 |

**可优化项（非阻断，建议后续迭代处理）：**

1. **清单项添加功能**：当前仅支持添加名称，可考虑增加目的地、分类等字段
2. **清单删除功能**：当前清单列表页和详情页均无删除按钮，可考虑添加
3. **清单项删除功能**：当前无单独删除清单项的入口，只能通过添加/切换状态操作

**总体评分：** 功能完整性 ★★★★☆ / 交互流畅度 ★★★★☆ / 边界健壮性 ★★★★☆
