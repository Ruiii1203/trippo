# tests 目录说明

这里用于存放 Trippo 的测试资料与验收记录。

## 可存放内容

- 页面验收用例
- 用户流测试脚本
- 手动测试记录
- 回归测试清单
- 截图对比记录

## 推荐起步方式

在项目早期，即使还没有正式测试框架，也建议先写两类内容：

1. `acceptance-v1.md`
   - 创建行程是否成功
   - 景点是否可加入心愿清单
   - 路线是否可按天排序
   - 打卡是否成功写入
   - 手账草稿是否能生成

2. `smoke-check.md`
   - 首页可打开
   - 路线页可切换
   - 打卡按钮可点击
   - 手账页可展示已有素材
