# Trippo 产品结构

这份文档用于把 PRD 中的产品结构翻译成更适合开发和页面拆分的形式。

## 主导航

基于当前产品形态验收反馈，Trippo 更适合采用 **3 个主入口**，更像常规 App、也更贴合用户心智：

- 行程
- 心愿单
- 旅行手账

说明：

- `心愿单` 与 `旅行手账` 都是核心功能，应当有底部一级入口
- `清单` 不再作为底部一级入口，而是 **内嵌在「心愿单」内部**（心愿单的一种组织方式）
- `发现` 属于后续版本能力（V3.0），当前不作为底部入口
- `我的/设置` 可作为后续可选入口，但当前阶段不应抢占三大主线入口的注意力

## 一级页面

### 行程

承接 V1.0 的核心使用路径。

包含：

- 行程列表
- 创建行程
- 行程详情
- 景点选择
- 路线规划
- 打卡记录
- 手账编辑
- 手账查看

### 发现

承接 V3.0 的个性推荐入口（后续版本）。

包含：

- 心情推荐
- MBTI 推荐
- 星座趣味推荐
- 推荐结果卡片
- 推荐转清单 / 转行程

### 心愿单

承接“长期收集 + 旅行前灵感积累”的核心场景，是 Trippo 的一级入口之一。

包含：

- 景点心愿
- 美食心愿
- 旅行地心愿
- 主题清单（V2.0 已实现的 List/清单能力，作为心愿单内部的一类内容）
- 心愿/清单转行程（例如把清单项或心愿点转为某趟行程的候选池）

心愿单同时需要支持两种视角：

- 全局入口：按行程分类查看全部内容 + 未关联行程的归类
- 行程内入口：只看当前行程相关的内容

### 旅行手账

承接“旅行后沉淀”的核心场景，是 Trippo 的一级入口之一。

包含：

- 手账列表（按行程分类 + 全部视图）
- 手账草稿（未完成/未归档）
- 手账详情/编辑（含模板）

同样需要支持两种视角：

- 全局入口：查看全部手账（可按行程分类）
- 行程内入口：只看当前行程对应手账

### 我的（后续可选）

承接账号与设置（后续版本或可选入口）。

包含：

- 账号信息
- 登录状态
- 云同步状态
- 独旅默认设置
- 权限与隐私设置
- 主题风格设置

## 核心页面关系

```text
行程列表
  -> 创建行程
  -> 行程详情
     -> 景点选择
     -> 路线规划
     -> 打卡
     -> 该行程心愿单
     -> 该行程手账

心愿单（全局）
  -> 按类型查看（景点/美食/旅行地/清单）
  -> 按行程查看（含未关联行程）
  -> 心愿/清单详情
     -> 关联到某个行程

旅行手账（全局）
  -> 按行程查看
  -> 草稿
  -> 手账详情/编辑（模板）
```

## 当前建议的前端拆分

如果后面开始写代码，建议以页面和领域对象双维度拆分：

- 页面层：`trips`、`discover`、`lists`、`profile`
- 领域层：`trip`、`spot`、`checkin`、`journal`、`list`

这样既方便按导航组织页面，也方便按业务对象抽离状态和组件。
