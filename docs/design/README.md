# 设计文档目录说明

这里用于存放正式进入设计与开发协同时最需要的界面和交互资料。

## 当前已存在的文件

- `product-structure.md`
- `user-flow.md`
- `ui-notes.md`
- `figma-links.md`

## 建议放入的内容

- `product-structure.md`：页面结构、主导航、模块关系
- `user-flow.md`：核心用户流，例如创建行程、打卡、生成手账
- `ui-notes.md`：页面交互规则、状态说明、组件行为
- `figma-links.md`：Figma、飞书文档、原型链接索引
- `visual-guidelines.md`：视觉风格、配色、留白、字体、图标风格
- `copywriting.md`：关键文案与语气
- `screenshots/`：关键页面截图

## 推荐使用方式

每次开始具体页面开发前，先在这里补齐对应页面的说明，而不是只把设计停留在对话里。

例如：

- 开发首页前，先写首页模块和卡片信息层级
- 开发路线页前，先写拖拽排序、空状态、路线过满提醒
- 开发独旅模式前，先写安全提醒边界和默认开关

## 原则

- 与 PRD 不重复写业务背景，重点补页面和交互细节
- 一页只解决一个主题，避免把所有内容塞进同一份长文档
- 尽量让文档能被开发直接引用，而不是仅供阅读
