# Trippo 项目上下文

这份文档是进入开发前的快速上下文入口，用于把 PRD 和版本规划压缩成一份更适合编码前阅读的摘要。

## 产品是什么

Trippo 是一个围绕“一次旅行”构建的移动端产品。它不是重交易的旅游平台，也不是泛日记工具，而是把一次旅行作为完整对象管理的旅行陪伴产品。

产品主线分为两层：

1. 核心闭环：行程规划、景点路线、打卡记录、旅行手账
2. 特色模块：主题清单、个性推荐、独自旅行模式

## 一句话定位

从“记录一次旅行的工具”升级为“懂你的旅行陪伴助手”。

## 核心用户

- 一年有两次以上自由行的普通旅行者
- 愿意拍照、打卡、记录，但不想在多个工具间来回切换
- 在出发前容易纠结“去哪”，在旅行后想低成本整理手账
- 对独旅、情侣、闺蜜、亲子等场景有明确差异化需求的人群

## 当前已确认产品边界

- 已实现（V1.0 + V2.0）：行程、路线、打卡、手账、主题清单、花费统计、手账模板
- 规划中（后续版本）：心情推荐、独旅模式、协作与 AI 增强
- 不做：机酒预订、重社区、达人内容平台、MVP 阶段的陌生人社交
- 地图策略：接入高德作为 POI 底座；导航跳第三方地图，不自建导航
- 数据策略：长期目标是云账号体系，支持多端同步和弱网本地缓存

### V1 + V2 当前实现形态（2026-07-07）

> 以下是当前代码仓库中的实际实现，与长期目标的差异已标注。

- **无后端**：纯前端实现，所有数据存储在浏览器 localStorage（key 前缀 `trippo_`）
- **无账号体系**：无登录、无多端同步，数据仅存在当前设备
- **无地图 API**：景点搜索使用本地 mock 数据池，不依赖高德/百度 API
- **无照片上传**：打卡的照片字段为占位，不涉及真实文件存储
- **已实现模块**：行程、景点搜索、心愿清单、每日路线、景点打卡、手账、主题清单、花费统计
- **技术栈**：React 18 + Vite 5 + TypeScript 5 + Zustand 4 + React Router v6
- **移动端优先**：最大宽度 480px 居中，支持安全区适配
- **UI 风格**：暖橙主色 `#FF7A45` + 纯白背景，完善的 token 体系

### 当前改版重点（信息架构 · 2026-07-08）

- 主入口更像常规 App：底部导航优先收拢为「行程 / 心愿单 / 旅行手账」
- 双入口规则：心愿单与旅行手账同时支持“全局入口”和“行程内入口（只看该行程相关内容）”
- 清单定位调整：清单能力内嵌在心愿单下（心愿单的一种内容组织方式），避免用户心智被拆散

## 版本顺序

### V1.0

先跑通核心闭环（长期目标，当前实现见上「V1 + V2 当前实现形态」）：

- 创建行程
- 景点搜索与心愿清单
- 每日路线规划
- 景点打卡
- 手账编辑与保存

### V2.0

补主题清单、花费完善、手账模板。**已完成。**

### V3.0

补“去哪”的决策能力，优先做心情推荐，再扩 MBTI 和星座。

### V4.0

补独旅模式，解决安全感、节奏、独拍和独处友好体验。

### V5.0

再做协作、AI 辅助、导出分享和智能排序。
