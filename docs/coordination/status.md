## 当前口径（不要跑偏）

- 产品定位：懂你的旅行陪伴助手
- 当前阶段：V2.0 验收与信息架构改版中
- 技术栈：React 18 + Vite + TypeScript + Zustand + React Router v6
- 入口层级（验收结论）：Trippo 更像常规 App，主入口应收拢为底部导航「行程 / 心愿单 / 旅行手账」
- 双入口规则：心愿单与旅行手账既要有“全局入口”，也要支持“从某个行程内进入只看该行程相关内容”
- 心愿单定位：心愿单是一级功能，内部按类型（景点/美食/旅行地/清单）与按行程两种维度组织；清单功能内嵌在心愿单之下
- 路线基于 Spot 自身字段（dayIndex + orderInDay）管理，不单独维护 DayPlan 实体
- 清单类型：人生/情侣/闺蜜/亲子/其他，清单项状态：想去/计划中/已完成
- 打卡分类：景点/美食/购物/交通/住宿/其他
- 手账模板：经典/旅行/可爱/极简
